# Ground-based Navigation Multipath Simulation and NavTCN Experiments

This repository contains the simulation code, model code, precomputed experimental data, and paper figures for the paper:

**Multipath Observation Correction for Ground-Based Navigation Using Post-Fit Residuals and Code-Minus-Carrier Consistency**

The repository is organized to reproduce the main results reported in the paper, including model comparison, NavTCN ablation, compound robustness analysis, and NavTCN complexity statistics.

## Repository Structure

```text
.
├── config.py                  # Global simulation and model configuration
├── main.py                    # Basic end-to-end ground-based navigation simulation entry
├── sim/                       # Signal, geometry, multipath, tracking, positioning, error modules
├── model/                     # Baseline and learning-based models: Huber, GRU, TCN, MSTCN, NavTCN
├── ablation/                  # Experiment scripts for the paper
├── outputs/                   # Original paper outputs with scripts' native file names
├── data/                      # Clean table-oriented copies of the paper data
├── figures/                   # Clean figure-oriented copies of the paper figures
└── requirements.txt           # Minimal Python dependencies
```

## Compared Methods

The main comparison includes seven methods:

- `baseline`: equal-weight least-squares positioning using raw code pseudoranges
- `Hatch`: Hatch code-carrier smoothing followed by weighted positioning
- `Huber`: robust iterative weighted least-squares positioning
- `GRU`: quality-weighting model using residual, CMC, and geometric features
- `TCN`: single-scale temporal quality-weighting model using the same features
- `MSTCN`: multi-scale temporal quality-weighting model using the same features
- `NavTCN`: proposed observation-domain pseudorange correction model

## Environment

Tested with Python 3.10+ on Windows. Install dependencies with:

```bash
pip install -r requirements.txt
```

`torch` is required for the deep-learning models. The code automatically selects the configured device. For CPU-only execution, set the device in `config.py` if needed.

## Notes on Randomness

The reported paper tables are based on fixed scene seed settings. Re-running the full stochastic scripts may produce slightly different values if configuration, random seeds, Python packages, or hardware settings are changed. The precomputed `data/` and `outputs/` files are therefore included to document the exact values used in the paper.

## Data Availability Statement

This repository contains the simulation source code, configuration files, experiment scripts, precomputed result tables, and figures used in the paper. No real-world ground-based navigation data are included; all released results are generated by the self-developed simulation platform described in the manuscript.
